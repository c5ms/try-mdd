# 原始需求记录

> AI 整理用户输入后的原始需求记录

---

## 领域名称
订单（Order）

## 核心实体
- 订单（Order）：业务核心实体
- 客户（Customer）：订单所属用户
- SKU：订单包含的商品明细

## 关联关系
- 客单关系：订单属于一个客户（1:1）
- 订单-SKU：每个订单包含多个 SKU（1:N）

## 状态定义
- 待支付、已支付、待发货、已发货、已完成、已取消

## 业务规则
- 支付后才能发货
- 发货后用户确认收货才算完成
- 待支付时可以取消订单
- 已发货后可以申请退款
- 订单金额超过 5000 元需要风控审核，审核通过才能支付
- 同一用户不能有超过 3 个待支付订单

## 用例场景
- 正常流程：创建 → 支付 → 发货 → 收货 → 完成
- 取消流程：待支付状态下取消
- 退款流程：已发货后申请退款
