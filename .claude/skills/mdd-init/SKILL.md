# MDD 项目初始化

本 skill **不自动触发**，需用户通过 `/m-init` command 调用。

---

## 核心理念

```
确认项目 → 设计角色 → 确定准则 → 初始化目录 → 用户可使用 /m-design 开始设计
```

---

## 输入

由 `/m-init` command 传入的已确认信息：
- 项目名称
- 项目定位（互联网/企业级）
- 项目背景
- 核心需求
- 参与角色
- 设计准则
- 核心领域列表

---

## 步骤

### Step 1：创建配置文件

#### 1.1 创建 config/rules.md

写入设计准则：

```markdown
# 设计准则

> AI 推演时必须遵循的设计规约

---

## 项目定位

- **类型**：{互联网/企业级}
- **复杂度**：{高/中/低}

---

## 设计准则

### 一、数据一致性

{准则内容}

### 二、并发处理

{准则内容}

### 三、权限控制

{准则内容}

### 四、业务合规

{准则内容}

---

## DDD 建模规约

{基础 DDD 规约内容}
```

#### 1.2 创建 config/roles.md

写入角色定义：

```markdown
# 角色定义

> 参与设计的角色及其专业视角

---

## 角色：{角色名}

**视角**：{视角}

**关注点**：{关注点}

**背景**：{背景}
```

---

### Step 2：创建核心领域目录

为每个核心领域创建目录：

```
domain/{domain}/
  .gitkeep
  model.md    # 空白模板
```

#### model.md 空白模板

```markdown
# {领域名称} 领域模型

> 等待推演...

---

使用 `/m-design {领域}` 开始设计。
```

---

### Step 3：创建其他目录

```
solution/
  .gitkeep

reflect/
  .gitkeep
```

---

### Step 4：创建项目文件

#### 4.1 创建 README.md

```markdown
# {项目名称}

## 项目简介

{项目背景描述}

## 核心领域

| 领域 | 类型 | 说明 |
|------|------|------|
| {领域1} | 核心域 | {说明} |
| {领域2} | 支撑域 | {说明} |

## 快速开始

使用 `/m-design {领域}` 开始设计领域模型。
```

#### 4.2 创建 .gitignore

```
# MDD 设计引擎忽略文件

# 临时文件
*.tmp
*.log

# IDE
.vscode/
.idea/

# 系统文件
.DS_Store
Thumbs.db
```

#### 4.3 创建 PROJECT.md

```markdown
# 项目定位

> 最后更新：{日期}

---

## 项目信息

- **项目名称**：{名称}
- **项目定位**：{定位}
- **业务领域**：{领域}

---

## 项目背景

{背景描述}

---

## 核心需求

{核心需求列表}

---

## 参与角色

| 角色 | 视角 | 背景 |
|------|------|------|
| {角色1} | {视角} | {背景} |

---

## 设计准则

{准则摘要}

---

## 核心领域

| 领域 | 类型 | 说明 |
|------|------|------|
| {领域1} | 核心域 | {说明} |
| {领域2} | 支撑域 | {说明} |
```

---

## 输出

完成后告知用户：

```
项目初始化完成：

项目信息文件：
- PROJECT.md    —— 项目定位说明
- README.md     —— 项目说明

配置文件：
- config/rules.md    —— 设计准则
- config/roles.md    —— 角色定义

核心领域：
- domain/{domain1}/model.md    —— {领域1}（空白，等待 /m-design）
- domain/{domain2}/model.md    —— {领域2}（空白，等待 /m-design）

其他：
- .gitignore    —— git 忽略文件
- solution/.gitkeep    —— 方案目录
- reflect/.gitkeep    —— 反思记录目录

下一步：使用 `/m-design {领域}` 开始设计领域模型。
```

---

## 约束

- 所有目录创建 `.gitkeep` 文件
- model.md 使用空白模板，等待 `/m-design` 填充
- rules.md 和 roles.md 写入用户确认的内容
- PROJECT.md 记录项目定位信息
