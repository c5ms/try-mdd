# 物料字典 设计方案

> 最后更新：2026-04-21

---

## 一、用户原话

> 领域名称：物料字典 用户可以创建物料字典。

> 物料字典需要包含：基本信息（编号、名称、规格）、物料分类、计量单位、供应商关联

> 物料字典关联领域：采购、生产

> 物料状态管理：简单状态（启用/停用）

---

## 二、AI 分析方案

### 术语表

| 术语 | 英文名 | 定义 | 关联术语 |
|------|--------|------|----------|
| 物料 | Material | 生产所需的原料、辅料、包装材料等基础物资 | 物料分类、计量单位、供应商 |
| 物料分类 | MaterialCategory | 物料的类别划分（面料、辅料、包装材料等） | 物料 |
| 计量单位 | UnitOfMeasure | 物料的计量标准（米、公斤、件等） | 物料 |
| 物料状态 | MaterialStatus | 物料的启用状态（启用/停用） | 物料 |

### 实体设计

#### 物料（聚合根）

| 属性 | 类型 | 必填 | 说明 |
|------|------|------|------|
| id | string | ✓ | 唯一标识 |
| code | string | ✓ | 物料编码（系统自动生成） |
| name | string | ✓ | 物料名称 |
| specification | string | | 规格型号 |
| categoryId | string | ✓ | 物料分类ID（聚合间引用） |
| unitId | string | ✓ | 计量单位ID（聚合间引用） |
| status | enum | ✓ | 物料状态（启用/停用） |
| remark | string | | 备注 |
| createdAt | datetime | ✓ | 创建时间 |
| updatedAt | datetime | | 更新时间 |

#### 物料分类

| 属性 | 类型 | 必填 | 说明 |
|------|------|------|------|
| id | string | ✓ | 唯一标识 |
| code | string | ✓ | 分类编码 |
| name | string | ✓ | 分类名称（面料、辅料、包装材料等） |
| parentId | string | | 父分类ID（支持层级） |
| status | enum | ✓ | 分类状态（启用/停用） |

#### 计量单位

| 属性 | 类型 | 必填 | 说明 |
|------|------|------|------|
| id | string | ✓ | 唯一标识 |
| code | string | ✓ | 单位编码（M、KG、PCS） |
| name | string | ✓ | 单位名称（米、公斤、件） |
| baseUnitId | string | | 基准单位ID（换算基准） |
| conversionFactor | decimal | | 换算系数 |

#### 物料供应商关联（内部实体）

| 属性 | 类型 | 必填 | 说明 |
|------|------|------|------|
| id | string | ✓ | 唯一标识 |
| materialId | string | ✓ | 物料ID（聚合内引用） |
| supplierId | string | ✓ | 供应商ID（聚合间引用） |
| isPreferred | boolean | | 是否首选供应商 |

### 聚合边界

**聚合：物料聚合**

- 聚合根：物料
- 内部实体：物料供应商关联（可多个）

**聚合：物料分类聚合**

- 聚合根：物料分类

**聚合：计量单位聚合**

- 聚合根：计量单位

### 聚合间引用

| 聚合A | 聚合B | 引用方式 |
|-------|-------|----------|
| 物料聚合 | 物料分类聚合 | categoryId |
| 物料聚合 | 计量单位聚合 | unitId |
| 物料聚合 | 供应商聚合 | supplierId（通过物料供应商关联） |
| 采购聚合 | 物料聚合 | materialId |
| 生产聚合 | 物料聚合 | materialId |

### 状态机

| 状态 | 编码 | 是否终态 | 说明 |
|------|------|----------|------|
| 启用 | ACTIVE | 否 | 物料可正常使用 |
| 停用 | INACTIVE | 是 | 物料停止使用 |

| 当前状态 | 触发事件 | 目标状态 | 前置条件 |
|----------|----------|----------|----------|
| 启用 | 停用 | 停用 | 无采购/生产订单在用 |
| 停用 | 启用 | 启用 | 无 |

### 业务规则

| 规则ID | 规则描述 | 来源 | 适用场景 |
|--------|----------|------|----------|
| R01 | 物料编码系统自动生成 | 系统架构师 | 创建物料 |
| R02 | 物料必须关联物料分类 | DDD规约 | 创建物料 |
| R03 | 物料必须关联计量单位 | DDD规约 | 创建物料 |
| R04 | 物料停用前需检查是否在采购/生产订单中使用 | 业务规则 | 停用物料 |

## 三、角色评审汇总

### 角色：系统架构师

| 维度 | 评价 | 意见 |
|------|------|------|
| 聚合边界 | ✓ | 物料、物料分类、计量单位独立聚合，边界清晰 |
| 聚合间引用 | ✓ | 通过ID引用，符合DDD规约 |
| 实体设计 | ✓ | 物料属性完整，支持多供应商关联 |
| 状态设计 | ✓ | 简单状态（启用/停用）符合需求 |
| 扩展性 | ✓ | 物料分类支持层级，计量单位支持换算 |

**补充建议**：
- 物料与供应商是多对多关系，已设计为独立的物料供应商关联实体 → 已采纳

### 角色：生产跟单员

| 维度 | 评价 | 意见 |
|------|------|------|
| 物料信息完整性 | ✓ | 物料编码、名称、规格、分类、单位覆盖完整 |
| 生产关联 | ✓ | 生产计划可引用物料ID |
| 物料状态 | ✓ | 停用物料不应出现在新生产计划中 |
| 物料分类 | ✓ | 面料、辅料分类便于生产管理 |

**补充建议**：
- 增加「物料图片」字段 → 待定
- 增加「物料安全库存」字段 → 待库存领域设计

### 角色：财务主管

| 维度 | 评价 | 意见 |
|------|------|------|
| 计量单位 | ✓ | 单位换算支持，便于成本核算 |
| 供应商关联 | ✓ | 物料可关联多个供应商，便于价格比较 |
| 成本相关 | △ | 物料价格信息待采购领域设计 |

**补充建议**：
- 物料价格信息应放在采购领域 → 已采纳
