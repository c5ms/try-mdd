## 🎯 执行路线（按GLM5建议）

### 第一周：破局（做出"顿悟时刻"）

**目标：**让人"哇"，找到人没想到但模型推出来的案例

**任务清单：**
```
❌ 任务8  → 做3个"推演对比"案例
  ↓
  产出：docs/blog-oschina.md 新增"推演对比"章节
  ↓
  产出：docs/case/ 目录存放真实案例日志
```

**关键产出：**至少1个"人没想到但模型推出来了"的案例

**验收标准：**
- 案例必须是真实的 `/m-design` 或 `/m-evolve` 日志
- 对比表格显示：人脑推演 vs MDD推演的差距
- 有"顿悟时刻"：MDD发现了人没想到的风险

---

### 第二周：信任（消除"PPT工程"怀疑）

**目标：**让人"信"，展示完整流程和公开细节

**任务清单：**
```
❌ 任务7  → 端到端案例（完整流程）
  ↓
❌ 任务11 → 公开 prompt 模板
  ↓
❌ 任务13 → 推演审计日志机制
```

**关键产出：**
- 完整端到端案例（从 `/m-init` 到 `/m-reflect`）
- 公开所有 skill 的 prompt 模板
- 推演过程可追溯（audit 日志）

**验收标准：**
- 案例包含所有命令的真实输出
- Prompt 模板有注释说明（为什么这么写）
- 审计日志记录：输入、推导链路、改了什么、为什么改

---

### 第三周：传播（让内容可自传播）

**目标：**让人"传"，制作传播载体

**任务清单：**
```
❌ 任务16 → 录3分钟"顿悟视频"
  ↓
❌ 任务17 → 做"MDD推演挑战"
  ↓
❌ 任务6  → 做"反面案例"（可选）
```

**关键产出：**
- 视频脚本：`docs/video/script.md`
- 推演挑战：`docs/challenge.md`（3个场景，让读者先想再看答案）
- 反面案例：`docs/case/failure-case.md`（如果当时有 MDD 会怎样）

**验收标准：**
- 视频时长 ≤ 3分钟
- 推演挑战有互动性（读者会主动分享结果）
- 反面案例有震撼力（损失比收益敏感10倍）

---

### 第四周：闭环（打通最后一公里）

**目标：**让人"用"，打通设计→编码

**任务清单：**
```
❌ 任务12 → Cursor/Copilot 打通指南
  ↓
❌ 任务14 → 回滚机制
  ↓
❌ 任务10 → 冷启动指南
```

**关键产出：**
- 打通指南：`docs/integration/cursor.md`
- 回滚命令：`/m-rollback`
- 新手指南：`docs/getting-started.md`

**验收标准：**
- model.md 可直接导入 Cursor 的 Rules for AI
- 回滚机制有自动备份
- 新手1小时内能跑通一个案例

---

### 之后：拔高（上升到"范式"）

**目标：**让行业用"MDD范式"这个词

**任务清单：**
```
❌ 任务15 → 测试用例集（可选）
  ↓
❌ 任务19 → KOL真实体验（可选）
  ↓
❌ 任务20 → 投准渠道（多平台发布）
```

**关键产出：**
- 推演准确性测试：`tests/` 目录
- KOL体验反馈：外部背书
- 多渠道发布：OSChina、V2EX、掘金、知乎、GitHub Trending

**验收标准：**
- 测试覆盖10个场景，准确率 ≥ 80%
- 至少1个KOL公开推荐
- 至少3个平台发布不同角度的文章
